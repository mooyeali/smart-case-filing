# 智能体升级第五阶段圆桌会议纪要

会议日期：2026-07-15

会议目标：评估第四阶段后的“可运营批量编目 agent”是否已经是完整且符合需求的智能体；如果不是，确定第五阶段开发范围并进入开发。

## 1. 当前状态复盘

第四阶段已经补齐运行时闭环能力：

| 能力 | 当前实现 | 验证证据 |
| --- | --- | --- |
| batch review 目录语义 | `--review-output <dir>` 控制批量 review 输出目录 | `tests/test_agent_batch_cli.py` |
| batch resume 持久化 | resume 后回写 output、manifest、review index | `tests/test_agent_partial_resume.py` |
| retry CLI | `--agent-retry-*` 参数传入 runner | `tests/test_agent_retry.py` |
| 模型配置预检 | `--agent-preflight` 不调用网络 | `tests/test_agent_preflight.py` |
| 人工复核决定 | `--review-decision` 写入 decision 文件并更新 manifest | `tests/test_agent_full_chain.py`、`tests/test_agent_run_manager.py` |

当前测试基线：

```text
python -m unittest discover -s tests
Ran 55 tests
OK
```

## 2. 结论

会议结论：当前系统已经是“可运营的批量编目 agent”，但还不是完整交付形态。需要开启第五阶段。

原因：

1. 缺少 run 级机器校验。现在 manifest、trace、output、review、decision 都能生成，但没有一个命令能证明这些产物彼此一致且完整。
2. 缺少可交付运行报告。用户仍需要手动打开 manifest 和各个目录查看状态分布、失败项、复核项和决定项。
3. 缺少最终验收门禁。全链路测试可以证明 fake workflow 通过，但不能直接验证一个已有 run directory 是否可归档。
4. 缺少针对缺失产物的明确诊断。例如 trace 丢失、review index 缺项、decision path 缺失时，现在没有统一错误列表。

## 3. 圆桌角色观点

### 产品负责人

用户需要的不只是“跑完”，还要知道“这批结果能不能交付”。第五阶段应提供一个验收命令，直接告诉用户 run 是否完整、哪些文件需要复核、哪些产物缺失。

### 架构负责人

不需要引入数据库或 UI。新增一个纯读取的 audit 模块即可：

```text
smart_case_filing/agent/audit.py
```

它读取 manifest 和文件系统，返回结构化验证结果，不改变 agent run 状态。

### 测试负责人

第五阶段测试应覆盖：

- 完整 run 返回 valid。
- 缺失 trace/output/review/decision 返回 issues。
- review index 覆盖所有 `NEEDS_REVIEW` 和 `FAILED`。
- CLI 可导出 Markdown 和 JSON 报告。

### 运维负责人

报告应适合归档和交付，至少包含：

- run id
- manifest path
- status counts
- review count
- decision count
- issue list
- per-file summary

## 4. 第五阶段建议范围

必做：

1. 新增 run audit helper。
2. 新增 CLI `--agent-validate-run <manifest-or-run-dir>`。
3. 新增 CLI `--agent-export-report <path>`。
4. 支持导出 Markdown 和 JSON 报告。
5. 更新文档和 fake run 报告。

暂不做：

1. Web UI。
2. 数据库持久化。
3. 递归目录处理。
4. 在线审批系统。
5. 自动修复缺失产物。

## 5. 会议决议

1. 当前功能还缺少最终交付验收门禁，因此不直接进入最终全链路验收完成。
2. 第五阶段目标定义为“可验收、可归档的批量 agent 运行产物”。
3. 第五阶段只新增 audit/report 能力，不改变核心预测流程。
4. 第五阶段完成后，才能把 run 目录作为可交付成果进行全链路验收。
