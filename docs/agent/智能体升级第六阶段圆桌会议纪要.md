# 智能体升级第六阶段圆桌会议纪要

会议日期：2026-07-16

会议目标：评估第五阶段后的可验收批量 agent 是否已经是完整且符合需求的智能体；如果是，开启全链路测试；如果仍有缺口，确定第六阶段开发范围。

## 1. 当前状态复盘

第五阶段已经具备可归档能力：

| 能力 | 当前实现 | 验证证据 |
| --- | --- | --- |
| run audit | `smart_case_filing/agent/audit.py` | `tests/test_agent_audit.py` |
| validate run CLI | `--agent-validate-run` | `tests/test_agent_audit.py` |
| audit report export | `--agent-export-report` | `tests/test_agent_audit.py` |
| 文档与报告 | README、使用说明、`test-reports/agent-phase-five-*` | Markdown 链接检查 |

当前测试基线：

```text
python -m unittest discover -s tests
Ran 61 tests
OK
```

## 2. 结论

会议结论：当前 agent 已经具备“可验收的批量运行产物”，但仍缺少一个独立、可重复、非 unittest 的全链路验收入口。因此先不开启最终人工全链路测试完成结论，而是进入第六阶段。

缺口：

1. fake full-chain 测试目前主要存在于 unittest 内部，不方便用户直接运行。
2. test report 是文档化样例，不是由一个可执行命令生成。
3. 真实模型 smoke 受环境配置影响，仍需要一个稳定的无模型验收按钮来证明 agent 编排、产物生成和 audit/report 能力。

## 3. 圆桌角色观点

### 产品负责人

用户需要一个“我按一下就知道这套 agent 能不能跑通”的命令。它不应该依赖真实模型，也不应该需要阅读测试代码。

### 工程负责人

建议新增 CLI：

```text
--agent-full-chain-test <output-dir>
```

它生成 deterministic fake run，并输出 summary。该命令应该覆盖：

- batch run
- completed / needs-review / failed
- review index
- review decision
- audit
- Markdown / JSON audit report

### 测试负责人

第六阶段验收应新增测试，直接调用 CLI，验证完整目录产物和 audit result。

### 运维负责人

输出目录必须可控。命令结束后应告诉用户 manifest、review index、audit report 的路径。

## 4. 第六阶段范围

必做：

1. 新增 fake full-chain runner helper。
2. 新增 CLI `--agent-full-chain-test <output-dir>`。
3. 生成完整 fake run 产物。
4. 自动运行 audit 并导出 Markdown / JSON 报告。
5. 更新文档和第六阶段报告。

暂不做：

1. 调真实模型。
2. Web UI。
3. 数据库持久化。
4. 递归目录处理。
5. 自动修复产物。

## 5. 会议决议

1. 当前系统离最终完整形态只差独立全链路验收入口。
2. 第六阶段目标定义为“可执行的无模型全链路验收命令”。
3. 第六阶段不改变核心预测逻辑，只新增 fake full-chain harness。
4. 第六阶段完成后，用户可以直接运行命令生成可审计产物。
